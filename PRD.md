## 1. 背景与问题

多语种市场的真实用户**不会规规矩矩只说一种语言**。海外华人、新加坡/瑞士/印度等多语国家用户、
以及习惯夹英文功能词的年轻用户，日常会自然地"中英混说"(code-switching)：

> "把 **air conditioning** 调到 20 度" · "把音量调 **louder**" · "导航去 **the office**"

现有车载语音助手大多按**单一语言**训练，遇到一句话里夹另一种语言就识别失败，
用户被迫重说或切换语言，体验断裂。这在出海车型上直接影响语音功能的使用率与口碑。

**一句话问题定义**：车机缺少一个能稳定理解"任意语言 / 混说"指令、并把它准确映射到车控功能的意图层，
也缺少一套能按语言维度量化这件事做得好不好的评测体系。

### 2.3 空档结论

没有任何一个方案同时打勾 **「混说 + 车载场景 + 可量化的多语种评测」**。
这三者的交叉点正是本项目的差异化定位，也正好是 PM（而非算法）该发力的地方。

## 3. 目标用户与场景

| 用户画像 | 典型场景 | 痛点 |
|----------|----------|------|
| 海外华人车主 | 中文为主，功能词习惯说英文("打开 navigation") | 助手听不懂夹杂的英文词 |
| 多语国家用户(新/瑞/印) | 一句话天然混两种语言 | 必须先选定语言，混说即失败 |
| 年轻本土用户 | 中文里夹流行英文词("放点 chill 的歌") | 被迫重说，体验断裂 |

**核心场景**：驾驶中(免提、注意力在路上)说出一条混说指令，助手一次听懂并执行，不要求重说或切语言。

## 4. 目标与非目标

**目标**
- 让"任意语言/混说"指令稳定映射到固定车控功能集。
- 提供按语言维度(中/英/混说)的质量看板，使"做得好不好"可量化。
- 云端不可用时有本地降级，保证语音功能基本可用。

**非目标(本期不做)**
- 不自研 ASR/TTS/LLM 模型(直接复用成熟底座)。
- 不做开放闲聊(非车控指令统一兜底到 `system.unknown`)。
- 不做多轮复杂对话(本期为单轮指令路由)。

## 5. 功能需求

| ID | 需求 | 优先级 | 验收标准 |
|----|------|:--:|------|
| F1 | 单轮指令路由到固定意图集 | P0 | 输出 intent + 槽位 + 置信度 |
| F2 | 混说(code-switching)识别 | P0 | mixed 样本准确率达标(见 §6) |
| F3 | 语言检测并标注 | P1 | 输出 detected_languages |
| F4 | 云端失败时规则降级 | P0 | 无 key/超时仍返回结果不崩溃 |
| F5 | 按语言维度评测看板 | P0 | 输出各语言准确率 + 明细 |
| F6 | 功能集可扩展 | P1 | 新增功能仅改 intents.py 一处 |

## 6. 指标设计（PM 核心交付物）

**北极星指标**：混说场景意图准确率(mixed intent accuracy)——直接代表差异化价值。

| 指标 | 定义 | 目标(示例) |
|------|------|:--:|
| 整体意图准确率 | 预测=期望 的比例 | ≥ 95% |
| **混说准确率** | mixed 子集准确率 | ≥ 90% |
| **语言差距(gap)** | 单语种均值 − 混说准确率 | ≤ 5pp |
| 兜底误触率 | 本应有效却被判 unknown 的比例 | ≤ 3% |
| 降级可用率 | 云端不可用时仍返回结果的比例 | 100% |

> 当前规则引擎基线：单语种 100%、混说 87.5%、gap 12.5pp ——
> 这正是引入 LLM 路由要收敛的差距，也是看板要持续盯的指标。

## 7. 关键取舍

- **买 vs 造模型**：不自研模型。模型层已饱和，PM 价值在场景定义、功能集设计、质量度量。
- **LLM vs 规则**：主用 LLM(覆盖混说与长尾)，规则仅作降级兜底——平衡效果与可用性。
- **混说优先级**：明确把混说设为 P0/北极星，而非"锦上添花"，因为它是差异化命脉。
- **评测先行**：先有按语言切分的评测集，再谈优化——避免"凭感觉调"。

## 8. 路线图

| 阶段 | 内容 |
|------|------|
| M1(已完成) | 双引擎路由 + 评测看板 demo，验证混说差距 |
| M2 | 评测集扩到数千条，按目标市场分语种维护；接真实 LLM 跑基线 |
| M3 | 增加方言与更多语种(日/德/阿)；接 ASR(如 SenseVoice)做端到端语音 |
| M4 | A/B：LLM 路由 vs 现网单语方案，看混说场景使用率与重说率变化 |

## 9. 风险

| 风险 | 缓解 |
|------|------|
| LLM 时延影响车机响应 | 设超时阈值 + 规则降级；高频指令走本地缓存 |
| 云端不可用 | F4 本地降级保底 |
| 长尾语言数据不足 | 用 LLM 零样本先覆盖，再按使用量回补评测集 |
| LLM 幻觉出非法 intent | 路由层强校验，非法 id 一律归 unknown |
